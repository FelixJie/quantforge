# 人形机器人投研看板 - 实现计划

## 1. 概述

创建一个专注于人形机器人主题的投研看板，包含研报信息获取、行业分析、相关概念股展示等功能。

## 2. 实现方案

### 2.1 后端增强

* 在 `research.py` 中新增获取行业/概念相关研报的 API

* 新增获取概念板块成分股的 API

* 新增获取龙虎榜的相关 API（已在 `signal_hub.py` 中）

### 2.2 前端看板视图

创建 `HumanoidRobotDashboard.vue` 视图，包含以下模块：

1. **头部概览**

   * 标题：人形机器人投研看板

   * 最近市场表现

   * 核心数据指标

2. **相关概念股**

   * 展示人形机器人概念相关股票列表

   * 包含实时行情、涨跌幅、PE、PB 等

   * 可点击查看详情

3. **机构研报**

   * 展示最新研报列表

   * 显示标题、发布机构、日期、评级

   * 可下载 PDF

4. **龙虎榜异动**

   * 展示近期上榜相关股票

   * 显示买入/卖出机构信息

5. **资金流向**

   * 展示概念板块资金流入/流出

   * 主力资金、融资融券数据

6. **市场情绪**

   * 使用图表展示相关指标

## 3. 技术架构

### 3.1 文件修改

1. `src/quantforge/api/routes/research.py` - 新增 API 端点
2. `web/src/views/HumanoidRobotDashboard.vue` - 新建看板视图
3. `web/src/main.js` - 添加路由
4. `web/src/App.vue` - 添加导航链接

### 3.2 数据依赖

* 研报数据源：东财 reportapi

* 实时行情：腾讯财经 API + mootdx

* 概念股：同花顺/东财概念板块

* 资金流向：东财数据中心

## 4. 实现步骤

1. 新增后端 API：获取概念板块成分股和研报
2. 创建前端看板视图组件
3. 添加路由配置和导航
4. 优化数据展示和样式
5. 测试和验证功能完整性

## 5. 潜在风险

* API 调用频率限制

* 数据结构变化

* 网络超时问题

## 6. 成功标准

* 看板可以正常加载和显示数据

* 所有功能按钮和链接正常工作

* 数据展示清晰、美观
